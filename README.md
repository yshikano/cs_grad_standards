# 達成度評価シート入力支援（GitHub Pages プロトタイプ）

`index.html` をそのまま GitHub Pages に置いて動かせる、クライアントサイド完結の試作です。

## この版で変えたこと

前版では、履修計画を作りながら 1 行ずつ Excel へ書き込める動線が残っていました。
この版では、**まずブラウザ内で履修計画ドラフトを作り、最後にだけ Excel へ一括反映する方式** に変更しています。

- Excel を読み込んでも、すぐには書き換えません
- 11〜20 行を UI 上の「履修計画ドラフト」として管理します
- 科目を選ぶたびに、**追加後の授業科目小計** と **各観点の必要点との差分** をシミュレーション表示します
- 履修計画ドラフト全体について、**授業科目小計 / 授業科目以外 / 見込み合計 / 差分 / 未達観点** を一覧表示します
- Excel への反映は、最後に **「履修計画を Excel に一括反映」** を押した時だけ行います
- ダウンロードは、ドラフトと Excel が一致している時だけ有効になります

## できること

- Excel テンプレート（達成度評価シート）をブラウザで読み込む
- 科目番号 / 授業名 / 英語名で検索する
- 1 科目ぶんの配点をプレビューし、必要なら手で微調整する
- 11〜20 行を「履修計画ドラフト」として管理する
- 選択中の科目を追加した場合の **小計・見込み合計・差分** を UI 上で確認する
- 現在の履修計画ドラフトについて、18 観点ごとの **必要点 / 授業科目小計 / 授業科目以外 / 見込み合計 / 差分** を表示する
- 読み込んだ Excel の 24〜26 行（授業科目以外）を見込み合計に自動加算する
- 「M中間評価」シートの 11〜20 行（A〜T 列）へ最後にまとめて書き込む
- 更新済みの Excel をダウンロードする

## 基本フロー

1. Excel を読み込む
2. 必要ならテンプレートに既に書かれている授業名や既存点数から、ドラフト計画を自動読込する
3. 科目を検索して、18 観点の配点を確認する
4. 「追加前シミュレーション」で、小計と必要点との差分を確認する
5. 問題なければ「履修計画ドラフトに追加 / 更新」を押す
6. 全科目が決まったら「履修計画を Excel に一括反映」を押す
7. その後に Excel をダウンロードする

## 重要な前提

公開されているカリキュラム・マップ PDF は **8 コンピテンス**（知の活用力 / マネジメント能力 / コミュニケーション能力 / チームワーク力 / 国際性 / 研究力 / 専門知識 / 倫理観）の配点です。

一方、添付の評価シートは **18 観点**（例: 知①, 知②, 研①, 研②, 研③ ...）で入力する形式です。

そのため、このプロトタイプでは次のどちらかで扱います。

1. `points18` がある科目は、その 18 観点の値をそのまま書く（厳密モード）
2. `points8` しかない科目は、アプリ側で 18 観点へ自動展開する（試作モード）

現在の `data/courses.sample.json` は、公開 PDF から読み取りやすい科目をサンプルとして入れたものです。
**公式どおりの厳密入力にしたい場合は、学内配布の「授業科目シート」等に対応する 18 観点マッピングへ差し替えてください。**

## GitHub Pages への置き方

1. このフォルダ一式を新しいリポジトリへ置く
2. リポジトリの Settings → Pages を開く
3. Branch を `main` / `/root` に設定する
4. 数十秒待つ
5. 公開 URL にアクセスする

## データ差し替え

`data/courses.sample.json` のフォーマットは次の通りです。

```json
{
  "meta": {
    "program": "情報理工学位プログラム（博士前期課程）"
  },
  "courses": [
    {
      "code": "0ALD512",
      "nameJa": "情報理工前期特別研究A",
      "nameEn": "Research in Computer Science A",
      "credits": 3,
      "total": 300,
      "points8": {
        "knowledge": 40,
        "management": 30,
        "communication": 40,
        "teamwork": 40,
        "internationality": 30,
        "research": 40,
        "specializedKnowledge": 40,
        "ethics": 40
      }
    }
  ]
}
```

厳密な 18 観点表を持っている場合は、`points18` を入れればアプリはそちらを優先します。

```json
{
  "code": "0ALD512",
  "nameJa": "情報理工前期特別研究A",
  "points18": [20,20,18,12,24,16,24,16,18,12,6,28,6,6,6,28,30,10]
}
```
