# 筑波大学達成度評価シート入力支援（非公式）

GitHub Pages でそのまま動く、筑波大学大学院向けの達成度評価シート入力支援アプリです。**非公式**ツールとして、システム情報工学研究群を中心に、大学院共通科目と人文社会科学研究群共通Mの一部科目も検索対象へ統合しています。

## この版で追加・修正した点

- **11〜14 行を固定欄化**
  - 情報理工前期特別研究 A〜D は **11〜14 行の固定欄** として扱います。
  - 通常科目は **15 行目以降** にしか入らないようにしています。
  - 旧版で 11〜14 行に通常科目が入っていた場合も、読込時に **15 行目以降へ自動で正規化** します。
- **省スクロール UI**
  - 画面上部に Excel / モード / 一括反映 / ダウンロードを集約
  - その下に **授業検索・授業科目 / 授業科目以外 / ドラフト・差分** の 3 タブを配置
  - 右カラムは **配点編集 / 時間割** をボタン切替にして、縦方向のスクロールを減らしました

- **時間割プレビューを簡素化**
  - 通常タームの時間割セルには **科目名だけ** を表示します。
  - **集中** タブを追加し、集中科目だけを一覧表示できます。
- **開閉できる UI**
  - 各カードに **縮小 / 展開** ボタンを追加しました。
  - 上部に **説明を省略 / 全パネルを縮小 / 全パネルを展開** ボタンを追加しました。
  - 使い方パネルも **表示 / 非表示** を切り替えられます。
- **時間割プレビューの読込強化**
  - `alternative-tsukuba-kdb` の `kdb-grad.json` を CDN / GitHub Raw など複数候補から取得
  - 読込済みの開講時限データは **ローカルキャッシュ** に保存
  - 自動取得できない場合に備えて、画面上から **`kdb-grad.json` の手動読込** も可能
  - 科目コード一致だけでなく、**科目名でも補助的に照合** します

## できること

- **3モード対応**
  - 履修計画
  - M中間評価
  - M最終評価
- 実際の様式 `Achievement-Evaluation-Sheet（M-2026-Enrollees）.xlsx` に合わせて、
  **M中間評価 / M最終評価** シートを読み込み・更新
- **M最終評価モード**では、中間評価ドラフトを初期値として流用可能
- システム情報工学研究群科目に加え、**大学院共通科目** と **人文社会科学研究群共通Mの一部科目** を **科目番号 / 授業名** で検索
- 公開カリキュラム・マップの **8 コンピテンス配点** から、B〜S列の **18 観点配点** を初期提案
- 18観点の点数は、**手動でそのまま修正可能**
- **授業科目以外** を追加可能
  - 論文・発表（最高点採用ルール）
  - 特許（300 × 貢献率）
  - 研究室WG
  - CSスペシャルワークショップ（CollaboTICS）
  - 学位プログラム外活動
  - TA（1コマ = 4点）
  - TOEIC / TOEFL（離散点）
  - 達成度自己点検レポート1 / 2
  - INFOSS情報倫理
  - APRIN eラーニングプログラム
- **UI 上で小計・合計・必要点との差分**を見ながら調整し、最後にだけ Excel へ **一括反映**

## 使い方

1. このフォルダ一式を GitHub Pages で公開します。
2. `index.html` を開きます。
3. 達成度評価 Excel を読み込みます。
4. 上部のモードで **履修計画 / M中間評価 / M最終評価** を選びます。
5. **授業検索・授業科目** タブで授業を追加します。
   - 情報理工前期特別研究 A〜D は 11〜14 行の固定欄です。
   - 通常科目は 15 行目以降に追加されます。
6. 必要なら **授業科目以外** タブで追加項目を入力します。
7. **ドラフト・差分** タブで、小計 / 合計 / 必要点との差分を確認します。
8. M中間評価または M最終評価モードで **「Excel に一括反映」** を押します。
9. 同期済みになったら更新済み Excel をダウンロードします。

## 時間割プレビューについて

- 開講時限データは `alternative-tsukuba-kdb` の大学院 KdB 参照データ `kdb-grad.json` を利用します。
- まず自動取得を試し、失敗した場合は画面右上の **「KdB JSON を手動読込」** から `kdb-grad.json` を読み込めます。
- 自動読込できたデータはローカルキャッシュにも保存するため、次回以降は再取得なしで使えることがあります。

## データについて

`data/courses.sample.json` は 2026 年度の博士前期課程向けの以下をもとに作っています。

- システム情報工学研究群の **公式開設授業科目一覧**
- 大学院スタンダードの **公開カリキュラム・マップ**

収録形式は大きく次の 3 種類です。


- 一部の大学院共通科目 / 他研究群科目は、公開カリキュラム・マップの **汎用コンピテンス数値** を、現在の評価シートの 5 汎用コンピテンスへ写像したデータとして追加しています。
- **○表記中心のカリキュラム・マップ** に備えて、`circle8` 形式の自動配分ルールを実装しています。
- `points8`: 8 コンピテンス配点が読める科目
- `rawCompetenceValues`: 公開 PDF で非ゼロ列だけ読める科目
- `creditsOnly`: 開設授業科目一覧で確認できるが、この JSON ではカリキュラム・マップ配点を同梱していない科目

`creditsOnly` の科目は、**単位数 × 100 点** を総点として、
`smart` / `required` / `even` のルールで 18 観点へ初期配点します。
その後、必要に応じて手動修正してください。

## GitHub Pages への配置

最低限必要なファイルは次です。

- `index.html`
- `app.js`
- `styles.css`
- `data/courses.sample.json`
- `.nojekyll`

このフォルダをそのままリポジトリへ置き、Pages の公開元に設定してください。

## 注意

公開カリキュラム・マップだけでは、18観点の **厳密値** までは与えられません。
このアプリでは、公開の 8 コンピテンス配点をもとに **18 観点の初期案** を作り、
最終的な点数は手動で調整できるようにしています。
