# 達成度評価シート入力支援（GitHub Pages プロトタイプ）

`index.html` をそのまま GitHub Pages に置いて動かせる、クライアントサイド完結の試作です。

## できること

- Excel テンプレート（達成度評価シート）をブラウザで読み込む
- 科目番号 / 授業名 / 英語名で検索する
- 1 科目ぶんの配点をプレビューし、必要なら手で微調整する
- 11〜20 行を「履修計画」として管理する
- 履修計画を作ると、18 観点ごとの見込み合計点・不足分・未達観点数を自動集計する
- 読み込んだ Excel の 24〜26 行（授業科目以外）も見込み合計に加える
- 「M中間評価」シートの指定行（A〜T 列）へ書き込む
- 履修計画を Excel にまとめて反映する
- 更新済みの Excel をダウンロードする
- 既に A 列へ入っている必修科目名（例: 情報理工前期特別研究A〜D）を自動照合して埋める

## 今回の追加点

前版では「1 行ずつ書き込む」支援が中心でした。
今回の版では、**履修計画そのものをアプリ内に持ち、合計点数をその場で自動計算する** ようにしています。

- 11〜20 行を履修計画の行として扱う
- 行ごとに「編集」「削除」ができる
- 履修計画の科目数 / 単位数 / 合計点を表示する
- 18 観点ごとに「必要点 / 履修計画 / 授業科目以外 / 見込み合計 / 不足分」を表示する
- Excel を読み込んだ時点で、既存行から履修計画を自動復元する

## 重要な前提

公開されているカリキュラム・マップ PDF は **8 コンピテンス**（知の活用力 / マネジメント能力 / コミュニケーション能力 / チームワーク力 / 国際性 / 研究力 / 専門知識 / 倫理観）の配点です。

一方、添付の評価シートは **18 観点**（例: 知①, 知②, 研①, 研②, 研③ ...）で入力する形式です。

そのため、このプロトタイプでは次のどちらかで扱います。

1. `points18` がある科目は、その 18 観点の値をそのまま書く（厳密モード）
2. `points8` しかない科目は、アプリ側で 18 観点へ自動展開する（試作モード）

現在の `data/courses.sample.json` は、公開 PDF から読み取りやすい科目をサンプルとして入れたものです。
**公式どおりの厳密入力にしたい場合は、学内配布の「授業科目シート」等に対応する 18 観点マッピングへ差し替えてください。**

## GitHub Pages への置き方

1. このフォルダ一式を新しいリポジトリへ置く
2. リポジトリの Settings → Pages を開く
3. Branch を `main` / `/root` に設定する
4. 数十秒待つ
5. 公開 URL にアクセスする

## データ差し替え

`data/courses.sample.json` のフォーマットは次の通りです。

```json
{
  "meta": {
    "program": "情報理工学位プログラム（博士前期課程）"
  },
  "courses": [
    {
      "code": "0ALD512",
      "nameJa": "情報理工前期特別研究A",
      "nameEn": "Research in Computer Science A",
      "credits": 3,
      "total": 300,
      "points8": {
        "knowledge": 40,
        "management": 30,
        "communication": 40,
        "teamwork": 40,
        "internationality": 30,
        "research": 40,
        "specializedKnowledge": 40,
        "ethics": 40
      }
    }
  ]
}
```

厳密な 18 観点表を持っている場合は、`points18` を入れればアプリはそちらを優先します。

```json
{
  "code": "0ALD512",
  "nameJa": "情報理工前期特別研究A",
  "points18": [20,20,18,12,24,16,24,16,18,12,6,28,6,6,6,28,30,10]
}
```
