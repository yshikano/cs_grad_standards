# 達成度評価シート入力支援

GitHub Pages でそのまま動く、筑波大学大学院システム情報工学研究群の達成度評価シート入力支援アプリです。

## 今回の版でできること

- **3モード対応**
  - 履修計画
  - M中間評価
  - M最終評価
- 実際の様式 `Achievement-Evaluation-Sheet（M-2026-Enrollees）.xlsx` に合わせて、
  **M中間評価 / M最終評価** シートを読み込み・更新
- **M最終評価モード**では、中間評価ドラフトを初期値として流用可能
- **情報理工前期特別研究A〜D** は **11〜14 行の固定欄** で手動配点
- 通常の受講科目は **15 行目以降** へ追加
- **11〜30行** を授業科目行として最初から使え、30行を超える場合もその下へ再配置
- システム情報工学研究群科目を**科目番号 / 授業名**で検索
- **時間割プレビュー** を見ながら、開講ターム・曜日時限の重なりを確認して履修計画を作成
- 公開カリキュラム・マップの **8コンピテンス配点** から、
  B〜S列の **18観点配点** を初期提案
- 18観点の点数は、**手動でそのまま修正可能**
- 他研究群科目は、元の **8コンピテンス** または **18観点** を入力して
  **1単位 = 100点** で自動初期配点
- **授業科目以外** を追加可能
  - 論文・発表（最高点採用ルール）
  - 特許（300 × 貢献率）
  - 研究室WG
  - CSスペシャルワークショップ（CollaboTICS）
  - 学位プログラム外活動
  - TA（1コマ = 4点）
  - TOEIC / TOEFL（離散点）
  - 達成度自己点検レポート1 / 2
  - INFOSS情報倫理
  - APRIN eラーニングプログラム
- **UI 上で小計・合計・必要点との差分**を見ながら調整し、
  最後にだけ Excel へ **一括反映**

## 使い方

1. このフォルダ一式を GitHub Pages で公開します。
2. `index.html` を開きます。
3. 達成度評価 Excel を読み込みます。
4. 上部のモードで **履修計画 / M中間評価 / M最終評価** を選びます。
5. 科目検索または他研究群科目の作成で授業科目を追加します。
   - 情報理工前期特別研究A〜Dは 11〜14 行の固定欄で配点します。
   - 通常科目は 15 行目以降へ追加します。
6. 授業科目以外も必要に応じて追加します。
7. 検索欄右の**時間割プレビュー**と下の集計表で、**小計 / 合計 / 差分 / 時限重複** を確認します。
8. M中間評価または M最終評価モードで **「Excel に一括反映」** を押します。
9. 同期済みになったら更新済み Excel をダウンロードします。

## データについて

`data/courses.sample.json` は 2026 年度の博士前期課程向けの以下をもとに作っています。

- システム情報工学研究群の**公式開設授業科目一覧**
- 大学院スタンダードの**公開カリキュラム・マップ**

収録形式は大きく次の 3 種類です。

- `points8`: 8 コンピテンス配点が読める科目
- `rawCompetenceValues`: 公開 PDF で非ゼロ列だけ読める科目
- `creditsOnly`: 開設授業科目一覧で確認できるが、この JSON では
  カリキュラム・マップ配点を同梱していない科目

`creditsOnly` の科目は、**単位数 × 100 点** を総点として、
`smart` / `required` / `even` のルールで 18 観点へ初期配点します。
その後、必要に応じて手動修正してください。

## 実装メモ

- 開講時限データは `alternative-tsukuba-kdb` の大学院 KdB 参照データ `kdb-grad.json` を利用します。
- まず `data/kdb-grad.json` の同梱ファイルを探し、見つからなければ CDN / GitHub Raw から取得します。
- 取り込んだ KdB のターム・曜日時限をもとに、右側の時間割表へドラフト科目と候補科目を重ねて表示します。
- Excel 更新はブラウザ上で `xlsx-populate` を使って行います。
- 行挿入に依存せず、授業科目・小計・授業科目以外・合計・不足分のブロックを再構成する方式です。
- 既存 Excel の M中間評価 / M最終評価を読み込んでドラフト化します。
- 最終評価シートが空に近い場合は、中間評価ドラフトを初期値として使えます。

## GitHub Pages への配置

最低限必要なファイルは次です。

- `index.html`
- `app.js`
- `styles.css`
- `data/courses.sample.json`
- `.nojekyll`

このフォルダをそのままリポジトリへ置き、Pages の公開元に設定してください。

## 注意

公開カリキュラム・マップだけでは、18観点の**厳密値**までは与えられません。
このアプリでは、公開の 8 コンピテンス配点をもとに **18観点の初期案** を作り、
最終的な点数は手動で調整できるようにしています。
