# 達成度評価シート入力支援

GitHub Pages でそのまま動く、筑波大学システム情報工学研究群の達成度評価シート入力支援アプリです。

## できること

- システム情報工学研究群の科目を科目番号・授業名で検索
- 公開カリキュラム・マップ由来のデータから 18 観点配点を自動提案
- `smart` モードで、現在の不足差分を見ながら自動配点
- 他研究群科目を 1 単位 = 100 点として扱い、元の 8 コンピテンスまたは 18 観点を入力して自動配点
- 履修計画ドラフトを UI 上で組み、差分を確認してから最後にだけ Excel を更新
- 10 科目を超える履修計画でも、Excel 側の授業科目欄を自動拡張して小計・合計・不足分を下へ再配置

## 使い方

1. このフォルダ一式を GitHub Pages で公開します。
2. ブラウザで `index.html` を開きます。
3. 達成度評価 Excel を読み込みます。
4. 研究群開講科目を検索するか、他研究群科目を作成します。
5. `smart` モードで不足差分を見ながらプレビューを確認します。
6. 履修計画ドラフトに追加し、並べ替えや削除で計画を整えます。
7. 最後に **「履修計画を Excel に一括反映」** を押します。
8. 同期済みになったら更新済み Excel をダウンロードします。

## データ形式

`data/courses.sample.json` は次の 2 種類を扱います。

- `points8`: 8 コンピテンスが完全に読める科目
- `rawCompetenceValues`: 公開 PDF から読める非ゼロ列のみを並べた科目

後者は、公開 PDF の行で 0 配点列が落ちることがあるため、アプリ側で現在の不足差分に最も合う位置へ自動配置してから 18 観点へ展開します。

## 実装メモ

- Excel 更新は `xlsx-populate` をブラウザで使って行います。
- `xlsx-populate` は行挿入を直接サポートしないため、このアプリでは授業科目行・小計・授業科目以外・合計・不足分のブロックを再構成して可変長にしています。
- 既存 Excel の「授業科目以外」欄の内容は保持したまま、下へ再配置します。

## GitHub Pages への配置

このフォルダをそのままリポジトリに置き、Pages の公開元をそのブランチ / フォルダへ設定してください。

最低限必要なファイルは次です。

- `index.html`
- `app.js`
- `styles.css`
- `data/courses.sample.json`

## 注意

公開カリキュラム・マップだけでは 18 観点の厳密値は得られません。厳密な授業科目シートが手元にある場合は、`points18` を持つ JSON に差し替えると、より厳密な運用に近づけられます。
