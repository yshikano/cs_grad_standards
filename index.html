5. この試作版の前提

本番化するなら、ここだけ差し替えればかなり実用になります。

  • 同梱の data/courses.sample.json は、公開カリキュラム・マップから読み取りやすい科目をサンプルとして入れています。
  • アプリ自体は points18（18 観点の厳密値）にも対応するよう作ってあります。公式の「授業科目シート」があれば JSON を差し替えるだけで厳密化できます。
  • いまは「授業科目」欄の入力支援が中心です。「授業科目以外」の論文・TOEIC・海外活動なども、同じ UI を追加すれば GitHub Pages 上で扱えます。
